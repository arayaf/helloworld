Workspace Studio & Automation 🤖

📅 อังคาร 21 ก.ค. 2569 ⏱ 30 นาที 🕓 16:00–16:30 น. 📍 ตึกอธิการบดี
🎯 วัตถุประสงค์
สร้าง automated workflow ด้วย Google Workspace Studio ได้โดยไม่ต้องเขียนโค้ด — เชื่อมต่อ Form, Sheets, Gmail, Calendar และ Drive เข้าหากัน เพื่อลดงานซ้ำ ๆ ในกระบวนงานธุรการจริง
✅ ผลลัพธ์การเรียนรู้
  • อธิบายได้ว่า Trigger คืออะไร และ Action คืออะไร
  • เข้าใช้ Google Workspace Studio และสร้าง workflow แรกได้
  • ประยุกต์ใช้ workflow กับกระบวนงานจริงในหน่วยงานตัวเองได้
  • รู้ความแตกต่างระหว่าง Workspace Studio (no-code) กับ Apps Script
⏱ แผนการสอน 30 นาที
เวลาหัวข้อรายละเอียด
0–5 นาที Hook: งานอะไรทำซ้ำทุกสัปดาห์? ถามกลุ่ม: "งานอะไรที่คุณทำซ้ำ ๆ ทุกสัปดาห์ที่อยากให้ระบบทำแทน?" — รวม 2-3 คำตอบ แล้วบอกว่าวันนี้จะสอนให้ทำได้จริง
5–10 นาที Workspace Studio คืออะไร เปิด workspace.google.com/products/studio — ชี้ให้เห็น: Trigger (เมื่อ...) → Action (ทำ...) เปรียบกับ "ถ้า...แล้ว..." ในชีวิตจริง · ไม่ต้องเขียนโค้ด · ใช้งานได้ด้วย Education Plus
10–18 นาที Demo Workflow #1 สด ผู้สอน demo: Form ถูกกรอก → ส่ง Email แจ้งอัตโนมัติ
เลือก Trigger "Form submitted" → Action "Send email" → ใส่ recipient, subject, body → Save → ทดสอบกรอก Form สด
18–24 นาที แบ่งกลุ่มลองทำ ผู้เรียนแต่ละคนเลือก workflow ที่ตรงกับงานตัวเอง (ดูหน้า 2-3) แล้วสร้างใน Studio — ผู้สอนเดินดูและช่วยเหลือ
24–27 นาที Demo Workflow #2 และ #3 ผู้สอน demo เร็ว ๆ: Sheet → Calendar event และ ไฟล์ใน Drive → แจ้งเตือน Chat (ไม่ต้องให้ผู้เรียนลองทำ เพื่อเห็นภาพ)
27–30 นาที Studio vs Script + สรุป Studio = ไม่มีโค้ด เหมาะกับงานทั่วไป · Apps Script = เขียนโค้ด ยืดหยุ่นสูง · "เริ่มจาก Studio ก่อน ถ้าทำไม่ได้ค่อยไป Script" · Homework: สร้าง workflow ใหม่ 1 อันก่อน S11
🔧 Workflow #1 — งานแผนและงบประมาณ
📊
นักวิเคราะห์นโยบายและแผน

รับแบบรายงานผลการดำเนินงาน → แจ้งหัวหน้าอัตโนมัติ

ปัญหาเดิม: เมื่อหน่วยงานส่งรายงานผลงานประจำเดือนผ่าน Google Form เจ้าหน้าที่ต้องคอยเช็คเองว่ามีใครส่งมาบ้าง แล้วค่อยโทรแจ้งหัวหน้า — เสียเวลาและพลาดบ่อย

Solution ด้วย Workspace Studio:

📋 Form ถูกกรอก
รายงานผลงาน
→
📊 Sheets
บันทึกข้อมูล
→
📧 Gmail
แจ้งหัวหน้าทันที

ขั้นตอนสร้างใน Workspace Studio:

  1. Trigger: "When a form is submitted" → เลือก Form "แบบรายงานผลการดำเนินงาน"
  2. Action 1: "Send an email" → To: หัวหน้าฝ่ายแผนฯ
  3. Subject: "📊 มีรายงานใหม่จาก: {{หน่วยงาน}}"
  4. Body: "หน่วยงาน {{ชื่อหน่วยงาน}} ส่งรายงานผลงานเดือน {{เดือน}} แล้ว กรุณาตรวจสอบใน Sheets: {{link}}"
  5. ทดสอบ: กรอก Form ทดสอบ → ตรวจ Email ว่าส่งถึงหัวหน้าจริง
⚠️ ผู้สอน: เตรียม Google Form "แบบรายงานผลงาน" ที่เชื่อม Sheets ไว้แล้วก่อนสอน (ต่อจาก S9)
🔧 Workflow #2 — งานบุคลากร (HRD)
👥
เจ้าหน้าที่บุคลากร

ลงทะเบียนอบรม → สร้าง Calendar event พร้อม invite อัตโนมัติ

ปัญหาเดิม: เมื่อมีผู้ลงทะเบียนอบรม เจ้าหน้าที่ต้องสร้าง Calendar invite ทีละคน ใส่ชื่อ-นามสกุล-อีเมล์เอง — ถ้าลงทะเบียน 20 คนก็ต้องทำ 20 ครั้ง

Solution ด้วย Workspace Studio:

📋 Form ลงทะเบียน
อบรม/สัมมนา
→
📊 Sheets
รายชื่อผู้เข้าอบรม
→
📅 Calendar
สร้าง event + invite
→
📧 Gmail
ยืนยันผู้เข้าร่วม

ขั้นตอนสร้างใน Workspace Studio:

  1. Trigger: "When a form is submitted" → เลือก Form "ลงทะเบียนอบรม"
  2. Action 1: "Create a calendar event" → ชื่อ event: "อบรม: {{หัวข้อ}}" · วันที่: {{วันที่อบรม}} · ผู้เข้าร่วม: {{อีเมล์ผู้ลงทะเบียน}}
  3. Action 2: "Send an email" → To: {{อีเมล์ผู้ลงทะเบียน}} · Subject: "✅ ยืนยันการลงทะเบียนอบรม {{หัวข้อ}}"
  4. ทดสอบ: กรอก Form ด้วยอีเมล์ตัวเอง → ตรวจ Calendar และ Gmail
💡 ประโยชน์จริง: อบรมครั้งหนึ่งมีผู้ลงทะเบียน 30 คน → ประหยัดเวลาสร้าง invite ได้ 30 ครั้ง × 3 นาที = 90 นาที/ครั้ง
🔧 Workflow #3 — หอสมุด / นักสารสนเทศ
📚
นักสารสนเทศ / บรรณารักษ์

ไฟล์รายงานใหม่ใน Drive → แจ้งทีมใน Google Chat อัตโนมัติ

ปัญหาเดิม: เมื่อนักสารสนเทศอัปโหลดรายงานสถิติการใช้ห้องสมุด / รายการหนังสือใหม่ / รายงานประจำปีลง Google Drive ทีมอื่นไม่รู้ว่ามีไฟล์ใหม่ ต้องส่ง Line หรือ Email แจ้งเองทุกครั้ง

Solution ด้วย Workspace Studio:

📁 ไฟล์ใหม่
ใน Shared Drive
→
💬 Google Chat
แจ้ง Space ทีมห้องสมุด
→
📧 Gmail
ส่งลิงก์ให้ผู้เกี่ยวข้อง

ขั้นตอนสร้างใน Workspace Studio:

  1. Trigger: "When a file is added to Drive" → เลือกโฟลเดอร์ "รายงานห้องสมุด" ใน Shared Drive
  2. Action 1: "Send a Chat message" → เลือก Space "ทีมสารสนเทศ" · ข้อความ: "📄 ไฟล์ใหม่: {{ชื่อไฟล์}} — {{ลิงก์ไฟล์}}"
  3. Action 2 (optional): "Send an email" → ส่งให้ผู้บริหารสำนักวิทยบริการ พร้อมลิงก์ไฟล์
  4. ทดสอบ: อัปโหลดไฟล์ทดสอบเข้าโฟลเดอร์ → ตรวจ Chat Space และ Gmail
💡 ต่อยอดได้: ปรับ Trigger เป็น "ทุกวันจันทร์ 8.00 น." แทน → ส่งสรุปรายสัปดาห์อัตโนมัติโดยไม่ต้องรอมีไฟล์ใหม่
⚠️ ต้องมี Google Chat Space ของทีมห้องสมุดที่ตั้งไว้แล้ว — ถ้ายังไม่มีให้สร้างก่อนเริ่ม workshop
⚖️ Workspace Studio vs Apps Script — เลือกใช้เมื่อไหร่?
หัวข้อ 🟢 Workspace Studio 🟡 Apps Script
ต้องเขียนโค้ดไหม ❌ ไม่ต้องเลย (No-code) ✅ ต้องเขียน JavaScript
ความยืดหยุ่น เลือกได้เฉพาะ trigger/action ที่มีในระบบ ทำได้ทุกอย่างที่คิดออก
เหมาะกับ งานทั่วไปที่ trigger ตรงกับที่มีใน Studio งานที่ต้องการ logic ซับซ้อน หรือ Studio ทำไม่ได้
เวลาตั้งค่า 5–10 นาที 30 นาที – หลายชั่วโมง
เรียนใน session S10 (วันนี้) S13–S16
📌 คำแนะนำ: เริ่มจาก Studio ก่อนเสมอ — ถ้า Studio ทำไม่ได้ค่อยไปเรียน Apps Script ใน S13
